# main-m-journal

## 项目简介

`main-m-journal` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为页面资料的集合与版本管理用途。  
所有页面均为独立 HTML 文件，适合用于内容存档、文档展示、静态页面托管等场景。

## 目录说明

```
main-m-journal/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的资源文件（如 CSS、JS、图片等）
├── archive/        # 历史版本或不再更新的页面备份
└── README.md       # 本说明文件
```

- `pages/`：主要页面目录，每个 HTML 文件对应一个独立页面。
- `assets/`：页面所需的公共资源，按需组织子目录。
- `archive/`：存放已归档的旧版本页面，便于追溯与回滚。

## 页面归档说明

- 所有页面均以 `.html` 后缀命名，文件名应简洁且具有辨识度。
- 每个页面应保持独立，不依赖外部不可控资源。
- 如需引用资源，建议将资源放置于 `assets/` 目录下，并使用相对路径。
- 页面内容可包含文本、样式、脚本等，但不包含任何营销或诱导性信息。

## 维护说明

- 本仓库由维护者定期更新，新增页面或修正现有内容。
- 欢迎通过 Issues 提出建议或报告问题。
- Pull Request 需遵循以下原则：
  - 新增页面请放入 `pages/` 目录。
  - 修改现有页面请保持原有结构与风格。
  - 避免引入外部跟踪代码、广告或恶意脚本。
- 仓库不保证页面内容的永久可用性，但会尽力保持历史版本可追溯。
